## 1.步骤
 - 登录服务器

 - 克隆项目代码到服务器
 - 安装项目依赖

### 2.3 克隆项目代码到服务器
在内存比较小的服务器（如1GB或2GB内存）中运行`npm install`,`npm run build`经常容易导致内存不足的问题然后系统挂掉。使用github actions可以避免这个问题。它的方案简单来说就是：
1. 在项目根目录创建 `.github/workflows/deploy.yml` 文件。
2. 配置它在每次推送代码时，自动帮你编译并把build文件夹推送到一个gh-pages或deploy分支。
3. 服务器操作：你只需要在服务器上克隆那个deploy分支即可。

```yml
name: Deploy Blog

on:
  push:
    branches:
      - main  # 当你向 main 分支推送代码时触发

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install and Build
        run: |
          npm install
          npm run build  # 这里执行打包命令，生成 build 文件夹

      - name: Deploy to Branch
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          folder: build    # 打包产物所在的目录
          branch: deploy   # 将产物推送到这个分支
```
